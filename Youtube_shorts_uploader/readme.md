# YouTube Shorts Auto-Uploader

Automate your YouTube Shorts uploads with this Python script optimized specifically for the Shorts format.

## What Are YouTube Shorts?

YouTube Shorts are vertical, short-form videos designed for mobile viewing:
- **Duration**: Up to 60 seconds
- **Aspect Ratio**: Vertical (9:16 preferred)
- **Resolution**: 1080x1920 recommended
- **Discovery**: Through Shorts feed, not traditional search
- **Format**: Quick, engaging, mobile-first content

## Libraries Used

### **google-auth** (v2.23.0+)
- **Purpose**: Core authentication library for Google APIs
- **Used in script**: 
  - `from google.auth.transport.requests import Request` - Refreshes expired OAuth tokens
  - `from google.oauth2.credentials import Credentials` - Manages OAuth 2.0 credentials
- **Why needed**: Handles OAuth 2.0 authentication for YouTube API access

### **google-auth-oauthlib** (v1.1.0+)
- **Purpose**: OAuth 2.0 integration for Google Auth Library
- **Used in script**:
  - `from google_auth_oauthlib.flow import InstalledAppFlow` - Runs OAuth authorization flow
- **Why needed**: Provides browser-based login flow and creates credentials from `client_secrets.json`

### **google-auth-httplib2** (v0.1.1+)
- **Purpose**: HTTP transport adapter for Google Auth
- **Used in script**: Automatically used by `googleapiclient` behind the scenes
- **Why needed**: Enables authenticated HTTP requests with automatic token refresh

### **google-api-python-client** (v2.100.0+)
- **Purpose**: Official Google API client library
- **Used in script**:
  - `from googleapiclient.discovery import build` - Creates YouTube API service instance
  - `from googleapiclient.http import MediaFileUpload` - Handles video uploads with resumable capability
  - `from googleapiclient.errors import HttpError` - Catches and handles API errors
- **Why needed**: Main interface to interact with YouTube Data API for uploading Shorts

### **Standard Python Libraries**
Built-in libraries that don't need installation:
- `os` - File system operations
- `json` - JSON file handling
- `pathlib.Path` - Modern path handling
- `time` - Delays for batch uploads
- `random` - Random jitter for retry logic
- `http.client` & `httplib2` - HTTP connection management

### File Specifications

#### **short.mp4** (or other formats)
- **Supported formats**: `.mp4`, `.mov`, `.webm`, `.avi`
- **Duration**: Maximum 60 seconds
- **Aspect Ratio**: 9:16 (vertical) - **CRITICAL**
- **Resolution**: 
  - Minimum: 720x1280
  - Recommended: 1080x1920
  - Maximum: 2160x3840 (4K)
- **File Size**: Typically under 50MB

#### **title.txt**
- **Content**: Short, catchy title
- **Character limit**: 100 characters
- **Auto-enhancement**: Script adds #Shorts if not present
- **Examples**:
  ```

  Watch Me Code This in 30 Seconds
  ```

#### **description.txt**
- **Content**: Brief description (keep it short!)
- **Character limit**: 5000 (but keep under 500 for Shorts)
- **Best practices**: 
  - 1-3 sentences maximum
  - Include call-to-action
  - Add relevant links if needed
- **Example**:
  ```

  ```

#### **hashtags.txt**
- **Content**: Trending and relevant hashtags
- **Character limit**: 500 characters total
- **Auto-enhancement**: #Shorts automatically added
- **Recommended**: 3-5 hashtags total
- **Examples**:

## Important Notes

### Testing Mode Limitation

‚ö†Ô∏è If your OAuth app is in "Testing" mode:
- Shorts will be locked as "Private"
- Cannot be changed to Public
- Solution: Publish your OAuth app OR manually change in YouTube Studio

### No Custom Thumbnails

- YouTube auto-generates Shorts thumbnails
- Uses frames from your video
- You cannot upload custom thumbnails
- Make sure your video has engaging frames throughout

### API Quotas

- **Daily quota**: 10,000 units
- **Short upload cost**: ~1600 units
- **Maximum Shorts per day**: ~6
- **Quota reset**: Midnight Pacific Time

### Upload Time

Shorts upload faster due to smaller file size:
- **10MB Short**: 30-60 seconds
- **25MB Short**: 1-2 minutes
- **50MB Short**: 2-4 minutes

### Shorts Discovery Timeline

- **Immediate**: Visible to subscribers
- **1-2 hours**: May appear in some Shorts feeds
- **6-24 hours**: Full algorithm processing
- **Peak performance**: Usually 2-3 days after upload

## Troubleshooting

### "Video must be vertical" warning
- Ensure your video is 9:16 aspect ratio
- Use video editing software to convert horizontal videos
- Tools: Adobe Premiere, Final Cut, CapCut, iMovie

### Shorts not appearing in Shorts feed
- Ensure #Shorts is in the title (script does this automatically)
- Video must be under 60 seconds
- Use vertical format (9:16)
- Wait 6-24 hours for processing

### Low views on Shorts
- First 3 seconds are critical - improve your hook
- Use trending hashtags
- Post at optimal times
- Engage with comments quickly
- Check YouTube Shorts analytics
- Be patient - Shorts can go viral weeks later

### "Quota exceeded" error
- You've uploaded too many Shorts today
- Wait until midnight Pacific Time
- Spread uploads throughout the day
- Consider uploading fewer Shorts (quality over quantity)
